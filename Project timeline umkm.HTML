03 · Titik Rilis

Detail Milestone

3 milestone
12 Oktober 2026

🏁 MVP Launch (Pilot)

  • Alur Kasir/POS berfungsi penuh, online maupun offline (Journey J2)
  • Produk, stok, buku kas, invoice/nota berjalan tanpa bug kritis
  • Minimal beberapa UMKM pilot sudah onboarding & transaksi nyata
  • Tidak ada bug kategori A/B (Transaksi, Stok) yang belum tertangani dari dokumen edge case
1 Februari 2027

🏁 Fase 2 Release

  • Multi-gudang, PO, hutang-piutang, laporan laba rugi/arus kas tersedia
  • Sinkronisasi offline penuh (bukan read-only lagi) sudah teruji sesuai edge case kategori C
  • RBAC granular aktif — karyawan tidak bisa akses menu di luar wewenang
  • Siap dibuka untuk pengguna umum, tidak terbatas pilot
26 April 2027

🏁 Fase 3 Release v1

  • Fitur AI (prediksi stok, prediksi penjualan, ringkasan bisnis) aktif untuk seluruh pengguna
  • Integrasi WhatsApp, email, payment gateway, dan minimal 1 marketplace berjalan stabil
  • Membership/loyalty, ekspor Excel, dan backup otomatis tersedia
  • Iterasi berikutnya (evaluasi PowerSync, wrap Capacitor bila perlu) berjalan berdasarkan feedback nyata setelah titik ini

05 · Jaring Pengaman

Alasan Penempatan Buffer

4 blok buffer
B1 · 28 Sep–11 Okt

Stabilisasi Pra-Peluncuran

Diletakkan sebelum milestone resmi (bukan sesudah), supaya hasil UAT pilot yang mungkin minta perubahan tidak menggeser tanggal launch.

HB · 21 Des–3 Jan

Musim Libur Akhir Tahun

Kapasitas tim realistis menurun karena cuti bersama. Dijadwalkan eksplisit di roadmap, alih-alih berpura-pura tim tetap produktif 100%.

B2 · 18–31 Jan

Hardening Sebelum Rilis Publik

Fase 2 dibuka untuk pengguna umum (bukan pilot terbatas lagi) — regression testing menyeluruh wajib sebelum eksposur bertambah.

B3 · 29 Mar–11 Apr

Ketergantungan Pihak Ketiga

Payment gateway, marketplace, dan WhatsApp Business API punya proses approval/verifikasi yang di luar kendali tim internal — buffer ini mengantisipasi keterlambatan tersebut.

RisikoDampak ke TimelineMitigasi
Verifikasi WhatsApp Business API / payment gateway molorMenunda go-live Sprint 14–15Mulai proses approval sejak awal S13, jangan tunggu sampai S14 dimulai.
Tim inti kecil, satu orang per peranSatu orang absen bisa hentikan satu workstreamDokumentasi teknis lengkap (Technical Spec) + sesi pairing berkala agar tidak ada silo pengetahuan.
