02 · Kriteria Lulus

Acceptance Criteria

7 modul kritis

1Transaksi POS — Online

GIVENkasir sudah login dan ada koneksi internet
WHENkasir menyelesaikan pembayaran (tunai/QRIS/split)
THENtransaksi tersimpan server <2 detik, stok berkurang real-time, struk tercetak, invoice number unik
Terkait: TC-POS-01, TC-POS-02, TC-POS-07

2Transaksi POS — Offline

GIVENtidak ada koneksi internet
WHENkasir menyelesaikan transaksi tunai
THENtersimpan ke local DB <200ms, badge Offline tampil, struk tetap cetak, masuk antrean sync ber-ID unik
Terkait: TC-POS-03, TC-EC-08

3Sinkronisasi Offline

GIVENada antrean transaksi offline tertunda (≤50)
WHENkoneksi internet kembali tersedia
THENsync otomatis <5 detik, tidak ada duplikasi, notifikasi ringkasan hasil ditampilkan
Terkait: TC-POS-04, TC-EC-04, TC-EC-09

4Kalkulasi Keuangan

GIVENada data transaksi & pengeluaran dalam rentang tanggal
WHENpengguna membuka laporan laba rugi/arus kas
THENangka akurat 100% sesuai data (toleransi Rp0), sesuai skema §6.2 Tech Spec
Terkait: TC-FIN-01, TC-FIN-02

5Hak Akses (RBAC)

GIVENuser dengan role tertentu login
WHENmencoba akses menu/endpoint di luar wewenang role-nya
THENditolak di UI (menu tersembunyi) DAN di API (403) — dua lapis proteksi, bukan salah satu saja
Terkait: TC-SEC-01, TC-SEC-02, TC-EC-10

6Performa di Device Rendah

GIVENdevice Android RAM 2GB, jaringan 3G Slow
WHENmembuka halaman Kasir pertama kali (cold start)
THENTime-to-Interactive <3 detik, Lighthouse Mobile ≥90, bundle JS awal <100KB gzip
Terkait: TC-PERF-01, TC-PERF-02, TC-PERF-03

7PWA Installability

GIVENpengguna membuka aplikasi lewat Chrome Android
WHENkriteria installability terpenuhi (manifest, service worker, HTTPS)
THENprompt "Add to Home Screen" muncul, app bisa dibuka standalone tanpa address bar
Terkait: §3.1 Tech Spec — @vite-pwa/sveltekit

Checklist DoD tambahan (modul non-kritis, cukup pass/fail)

  • Produk & kategori: CRUD lengkap, barcode/QR ter-generate valid dan bisa di-scan ulang
  • Manajemen stok: penyesuaian manual tersimpan dengan catatan alasan
  • Purchase Order: draft tersimpan, status berubah sesuai alur (draft → dikirim → diterima)
  • CRM: data pelanggan tersimpan, riwayat transaksi per pelanggan bisa dilihat
  • Kalkulator harga/margin/BEP/ROI: hasil hitung sesuai rumus, tanpa perlu simpan data
  • Ekspor PDF/Excel: file terbuka valid di aplikasi standar (Adobe Reader, Excel/Sheets)
  • Reminder WhatsApp: pesan terkirim dengan template & data pelanggan yang benar
  • Backup otomatis: file backup dapat dipulihkan (restore) tanpa kehilangan data
